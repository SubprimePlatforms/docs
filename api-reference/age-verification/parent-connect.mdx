```
POST /v1/age-verification/parent-connect
```

Initiates a Parent Connect flow where a parent or guardian verifies their identity and sets boundary controls on behalf of a minor. The platform defines what boundaries are available via `boundary_options`.

## Request

### Headers

| Header         | Required | Description        |
| -------------- | -------- | ------------------ |
| `x-api-key`    | Yes      | Your API key       |
| `Content-Type` | Yes      | `application/json` |

### Body parameters

| Parameter            | Alias               | Type    | Required | Description                                                          |
| -------------------- | ------------------- | ------- | -------- | -------------------------------------------------------------------- |
| `first_name`         | `firstName`         | string  | Yes      | Child's first name                                                   |
| `last_name`          | `lastName`          | string  | Yes      | Child's last name                                                    |
| `email`              | ---                 | string  | Yes      | Child's email address                                                |
| `phone`              | ---                 | string  | Yes      | Child's phone in E.164 format                                        |
| `parent_connect`     | `parentConnect`     | boolean | Yes      | Must be `true`                                                       |
| `age_band`           | `ageBand`           | string  | No       | Age band (e.g. `"13-17"`). Defaults to `"13-17"`                     |
| `boundary_options`   | `boundaryOptions`   | object  | No       | Platform-defined boundary configuration (see below)                  |
| `external_id`        | `externalId`        | string  | No       | Your internal reference ID                                           |

### `boundary_options` object

| Field                  | Type   | Description                                               |
| ---------------------- | ------ | --------------------------------------------------------- |
| `eula_acceptance`      | object | `{ required: boolean, eula_url: string }` EULA config     |
| `feature_restrictions` | array  | Feature boundary options the parent can toggle             |
| `content_restrictions` | array  | Content boundary options the parent can toggle             |

Each restriction in the arrays follows this schema:

| Field      | Type    | Required | Description                                             |
| ---------- | ------- | -------- | ------------------------------------------------------- |
| `key`      | string  | Yes      | Unique identifier for this boundary                     |
| `label`    | string  | Yes      | Human-readable label shown to the parent                |
| `type`     | string  | Yes      | `toggle`, `number`, `minutes`, or `select`              |
| `default`  | any     | No       | Default value                                           |
| `required` | boolean | No       | If `true`, parent must set this boundary                |
| `min`      | number  | No       | Minimum value (for `number` and `minutes`)              |
| `max`      | number  | No       | Maximum value (for `number` and `minutes`)              |
| `options`  | array   | No       | Allowed values (for `select`)                           |

## Response

### 200 --- Success

| Field                    | Type   | Description                                              |
| ------------------------ | ------ | -------------------------------------------------------- |
| `parent_connect_id`      | string | Unique ID for this Parent Connect flow                   |
| `child_session_id`       | string | The child's verification session ID                      |
| `status`                 | string | `PENDING`                                                |
| `age_band`               | string | Age band (e.g. `"13-17"`)                                |
| `boundary_options`       | object | Echoed back with feature/content grouping                |
| `consent_expires_at`     | string | When the consent will expire (180 days)                  |
| `session_url`            | string | URL for the child's age verification session             |
| `parent_verification_url`| string | `null` until invite is sent                              |
| `links`                  | array  | HATEOAS links for next actions                           |

### Error responses

| Status | Description                              |
| ------ | ---------------------------------------- |
| `400`  | Invalid body or `parent_connect !== true`|
| `401`  | Missing or invalid `x-api-key`           |
| `402`  | Insufficient token balance               |

## Next steps

After creating a Parent Connect request:

1. **Invite the parent** --- `POST /v1/age-verification/parent-connect/{id}/invite` with the parent's phone number or email
2. **Poll for status** --- `GET /v1/age-verification/parent-connect/{id}` returns the current status and, once consent is given, the `child_attestation_id`
3. **Submit consent** --- `POST /v1/age-verification/parent-connect/{id}/consent` (called by the parent after verification)
4. **Query boundaries** --- `GET /v1/age-verification/parent-connect/boundaries/{attestation_id}` to look up the current boundary settings

<Info>
  The parent can update boundaries at any time after consent via
  `PUT /v1/age-verification/parent-connect/boundaries/{attestation_id}`.
  No re-verification is required.
</Info>
