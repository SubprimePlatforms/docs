```
GET /v1/regulations/{country_code}
```

Returns the age verification regulatory configuration for a given country, including consent ages, allowed/banned verification methods, enforcement status, and applicable legislation.

Use `GET /v1/regulations` (no country code) to list all 101+ mapped jurisdictions.

## Request

### Headers

| Header      | Required | Description  |
| ----------- | -------- | ------------ |
| `x-api-key` | Yes      | Your API key |

### Path parameters

| Parameter      | Type   | Required | Description                                     |
| -------------- | ------ | -------- | ----------------------------------------------- |
| `country_code` | string | Yes      | ISO 3166-1 alpha-2 code (e.g. `GB`, `US`, `AU`) |

### Example request

## Response

### 200 --- Success

| Field                       | Type    | Description                                                     |
| --------------------------- | ------- | --------------------------------------------------------------- |
| `country_code`              | string  | ISO country code                                                |
| `country_name`              | string  | Human-readable country name                                     |
| `region`                    | string  | Geographic region                                               |
| `tier`                      | string  | `P0` (active enforcement), `P1`, `P2` (proposed), `P3` (limited)|
| `default_consent_age`       | integer | Default digital consent age                                     |
| `consent_ages`              | object  | Category-specific ages (see below)                              |
| `legislation`               | array   | Primary legislation references                                  |
| `regulator`                 | string  | Regulatory body name                                            |
| `enforcement_status`        | string  | `active`, `pending`, `proposed`, `limited`, or `none`           |
| `allowed_methods`           | array   | Permitted verification methods                                  |
| `banned_methods`            | array   | Prohibited methods (e.g. France bans `face-recognition`)        |
| `parental_consent_required` | boolean | Whether parental consent is needed for minors                   |
| `parental_consent_age`      | integer | Age below which parental consent is required                    |
| `outright_ban`              | boolean | Whether minors are banned entirely (no consent option)          |
| `max_penalty`               | string  | Maximum regulatory penalty                                      |
| `notes`                     | string  | Additional regulatory context                                   |

### `consent_ages` object

| Field             | Type           | Description                                           |
| ----------------- | -------------- | ----------------------------------------------------- |
| `general`         | integer        | General digital consent age                           |
| `social_media`    | integer        | Social media minimum age                              |
| `gaming`          | integer        | Online gaming / gambling age                          |
| `adult_content`   | integer        | Adult content access age                              |
| `alcohol_tobacco` | integer/object | Alcohol/tobacco age (may be `{ beer: 16, spirits: 18 }`) |

### Error responses

| Status | Description                                  |
| ------ | -------------------------------------------- |
| `401`  | Missing or invalid `x-api-key`               |
| `404`  | No regulatory data for the given country code|
