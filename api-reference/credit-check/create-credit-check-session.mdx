---
title: Create Credit Check Session
description: Create a standalone hard or soft credit check session and invite the applicant to consent
api: POST /v1/credit-check/{variant}
---

```
POST /v1/credit-check/hard
POST /v1/credit-check/soft
```

Creates a standalone credit check session and optionally sends email and SMS invitations to the applicant.

The `{variant}` path segment controls the inquiry type:

| Variant | Session type          | Effect on applicant's credit | Response depth                                                     |
| ------- | --------------------- | ---------------------------- | ------------------------------------------------------------------ |
| `hard`  | `credit-check-hard`   | May impact credit score      | Full credit insights (trades, collections, bankruptcies, AI summary) |
| `soft`  | `credit-check-soft`   | No impact on credit score    | Credit score only                                                   |

Unlike `/v1/sessions`, credit check sessions do **not** accept a `workflow_id` тАФ they are standalone. The applicant completes a consent step in the verify app (where they confirm personal details and, where applicable, provide SIN). The Equifax inquiry is then executed server-side automatically.

## Request

### Headers

| Header         | Required | Description        |
| -------------- | -------- | ------------------ |
| `x-api-key`    | Yes      | Your API key       |
| `Content-Type` | Yes      | `application/json` |

### Path parameters

| Parameter | Type   | Required | Description                                    |
| --------- | ------ | -------- | ---------------------------------------------- |
| `variant` | string | Yes      | Either `hard` or `soft` тАФ sets the inquiry type |

### Body parameters

Both camelCase and snake_case parameter names are accepted. If both are provided for the same field, the camelCase value takes priority.

| Parameter           | Alias             | Type    | Required | Description                                                                                          |
| ------------------- | ----------------- | ------- | -------- | ---------------------------------------------------------------------------------------------------- |
| `first_name`        | `firstName`       | string  | Yes      | Applicant's first name                                                                               |
| `last_name`         | `lastName`        | string  | Yes      | Applicant's last name                                                                                |
| `email`             | тАФ                 | string  | Yes      | Applicant's email address                                                                            |
| `phone`             | тАФ                 | string  | Yes      | Applicant's phone number in E.164 format (e.g. `+15192223333`)                                       |
| `external_id`       | `externalId`      | string  | No       | Your internal reference ID for this session                                                          |
| `send_email_invite` | `sendEmailInvite` | boolean | No       | Send an email invitation to the applicant. Defaults to `true`                                        |
| `send_phone_invite` | `sendPhoneInvite` | boolean | No       | Send an SMS invitation to the applicant. Defaults to `true`                                          |
| `redirect_url`      | `redirectUrl`     | string  | No       | HTTPS URL to redirect the applicant to after the session ends. Must be a valid HTTPS URL             |

## Response

### 200 тАФ Success

| Field         | Type   | Description                                                        |
| ------------- | ------ | ------------------------------------------------------------------ |
| `id`          | string | Unique identifier for the created session                          |
| `session_url` | string | URL the applicant opens to complete consent and information entry  |
| `type`        | string | Either `credit-check-hard` or `credit-check-soft`                  |
| `externalId`  | string | Your external ID (only returned if provided in the request)        |
| `links`       | array  | Associated admin console and session detail links                  |

### Error responses

| Status                  | Description                                                   |
| ----------------------- | ------------------------------------------------------------- |
| `400 Bad Request`       | Invalid request body тАФ check required fields and phone format |
| `401 Unauthorized`      | Missing or invalid API key                                    |
| `402 Payment Required`  | Insufficient token balance or no credit-check coverage         |
| `404 Not Found`         | Organization not found                                        |
| `429 Too Many Requests` | Rate limit exceeded                                           |

## How the session completes

1. Applicant opens the `session_url` and is guided through the credit check consent flow in the verify app
2. They confirm identity fields (name, DOB, address) and, for hard inquiries, provide SIN
3. On submission, the server runs the appropriate Equifax inquiry:
   - **Hard** тЖТ returns full parsed credit report plus AI-generated credit insights
   - **Soft** тЖТ returns the credit score only
4. Use [`GET /v1/sessions/{id}`](/api-reference/sessions/retrieve-session) to fetch the completed session and its analysis

<Info>
  The SIN (Social Insurance Number) collected during the consent step is stored encrypted for the duration of the inquiry and is automatically purged after the check completes. It is never returned by any API.
</Info>
